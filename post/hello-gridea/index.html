使用 Gridea + Github Pages 搭建我的博客

2020-01-09 4 min read # Gridea

兵马未动,工具先行

我有一个毛病,想要做一件事儿时,总是执着于工具上的研究.比如写博客用到的工具。在尝试了各种各样的工具之后，「此时从萌生想法到确定工具已过去两年，拖延症晚期了可以说。」最终确定了使用 Gridea 配合 Github 来搭建。

✍️ Gridea 一个静态博客写作客户端。你可以用它来记录你的生活、心情、知识、笔记、创意... ...

Github
Gridea 主页

特性👇

📝 你可以使用最酷的 Markdown 语法，进行快速创作

🌉 你可以给文章配上精美的封面图和在文章任意位置插入图片

🏷️ 你可以对文章进行标签分组

📋 你可以自定义菜单，甚至可以创建外部链接菜单

💻 你可以在 Windows，MacOS 或 Linux 设备上使用此客户端

🌎 你可以使用 𝖦𝗂𝗍𝗁𝗎𝖻 𝖯𝖺𝗀𝖾𝗌 或 Coding Pages 向世界展示，未来将支持更多平台

💬 你可以进行简单的配置，接入 Gitalk 或 DisqusJS 评论系统

🇬🇧 你可以使用中文简体或英语

🌁 你可以任意使用应用内默认主题或任意第三方主题，强大的主题自定义能力

🖥 你可以自定义源文件夹，利用 OneDrive、百度网盘、iCloud、Dropbox 等进行多设备同步

选择

我这里使用的是 Github Pages ，Gridea 的客户端文档编辑器感觉不是很好用，我使用的是 Typora 来写文章。评论系统使用 Gitalk，图床工具使用 PicGo，和 Typora 配合起来使用很舒服，直接在编辑器粘贴自动上传图片。

准备

前往Gridea 官网下载安装包安装，Gridea 的操作使用不再赘述。Gridea 还可以设置不同的主题，主题选择可以去这里选择。

发布

如果没有 Github 账户，注册一个 Github 账户，登录 GitHub 之后，点击这里创建一个 Token，勾选上 repo 的相关权限即可。生成之后记得把 Token 复制到你的本地，后边要用，因为一旦关闭网页将不能再看到它。

创建仓库，存放构建后的网站文件。创建一个公开仓库，名为用户名.github.io。（将用户名替换为你的 GitHub 用户名），我的是 sgaxun.github.io。

进入 Gridea 进行远程设置。按照下图的进行设置，然后点击「检测远程链接」，进行配置检查。

同步。点击 Gridea 中的 同步按钮即可。初次部署等待几分钟之后就可以去浏览器访问了，访问地址默认是 sgaxun.github.io。

自定义域名

Gridea 支持自定义域名，在远程设置中提供了 CNAME 选项。我在 name.com 购买了一个.me 的域名，感觉非常适合个人博客使用。在 DNS 记录中添加一条指向 sgaxun.github.io。如图

在 Gridea 的远程设置中 CNAME 添加 blog.sgaxun.me。以后访问博客使用 blog.sgaxun.me 即可。

Gitalk

添加 Gitalk 评论，需要创建 GitHub Application，如果没有 点击这里申请

创建成功后会得到一个 Client Id 和 Client Secret。将它们配置到 Gridea 中的评论配置里面。

关于写作

Markdown 编辑器我还是更喜欢使用 Typora，Typora 可是说是一个非常纯粹的 Markdown 编辑器了，所见即所得，使用起来非常舒服。图床本来我使用的是七牛云，免费额度够用了，最近在少数派读到一篇文章图床搭配 PicGo：打造高效的图片处理工作流。可以在码云创建一个 Project 作为图床仓库使用，图床工具 PicGo 可以通过安装插件的方式来使用 Gitee 图床。
