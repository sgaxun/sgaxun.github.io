服务器常用软件安装记录之一

2020-01-01 6 min read # Git # Nginx # Node # Java # Linux

背景

日常在服务器安装软件的场景发生的不是太多，每次安装总是记不住步骤,总是需要上网查，并不能做到加深记忆,因此产生了记录一下自己日常的操作习惯的想法，并为此研究了一下自己搭建博客，发现利用Github可以在不自己购买服务器的情况下非常简单的搭建起来。下篇文章介绍博客的搭建。

本文安装的软件都是基于Centos 7

Git

yum安装

一行命令安装

yum install git

源码编译安装

  1. 下载安装包

前往官网下载git安装包git下载,我下载了最新版git-2.20.1.tar.gz,下载至/usr/local/,将安装包解压

tar -zxvf git-2.20.1.tar.gz
  1. 安装依赖
yum install curl-devel expat-devel gettext-devel openssl-devel
zlibdevel gcc-c++ perl-ExtUtils-MakeMaker
  1. 编译安装Git

进入到对应目录，执行配置、编译、安装命令即可

cd git-2.20.1/
make configure
./configure --prefix=/usr/local/git
make profix=/usr/local/git
make install
  1. Git加入坏境变量

编辑配置文件

vim /etc/profile

尾部加入Git的bin路径配置即可

export GIT_HOME=/usr/local/git
export PATH=$PATH:$GIT_HOME/bin

JDK

准备安装包

首先去oracle官网下载jdk安装包,JDK下载地址,我下载的的是jdk-8u251-linux-x64.tar.gz,安装包上传至/usr/local目录下。

卸载openjdk,如果有的话

如果系统带有OpenJDK,首先进行卸载.
查找已经安装的OpenJDK

rpm -qa | grep java

然后将查找到的java开头的安装包全都卸载

yum -y remove java-1.7.0-openjdk-1.7.0.141-2.6.10.5.el7.x86_64 
yum -y remove java-1.8.0-openjdk-1.8.0.131-11.b12.el7.x86_64
...省略...

创建目录并解压

  1. 解压,将准备好的安装包解压到当前目录
cd /usr/local/
tar -zxvf /usr/local/jdk-8u251-linux-x64.tar.gz -C ./

解压完之后,/usr/local/目录中会出现一个jdk1.8.0_251的文件夹

  1. 重命名
    重命名jdk1.8.0_251文件夹为java
mv jdk1.8.0_251/ java/

环境变量

编辑/etc/profile文件,在文件末尾加入如下内容:

JAVA_HOME=/usr/local/java
CLASSPATH=$JAVA_HOME/lib/
PATH=$PATH:$JAVA_HOME/bin
export PATH JAVA_HOME CLASSPATH

执行如下命令让环境变量生效

source /etc/profile

输入如下命令验证JDK安装结果

java -version
javac

Node

下载

前往node官网下载最新版安装包,下载地址,我下载的是node-v11.4.0.tar.gz

创建目录并解压

cd /usr/local/
tar -xJvf node-v11.4.0.tar.gz C ./
mv node-v11.4.0/ node/

配置系统环境变量

编辑~/.bash_profile⽂件，在⽂件末尾追加如下信息

# Nodejs 
export PATH=/usr/local/node/bin:$PATH

刷新环境变量

source ~/.bash_profile

使用node -v来检查安装结果

Nginx

yum安装

yum安装方式非常简单,一条命令即可实现安装和卸载

yum -y install nginx
yum remove nginx

yum安装后配置文件的路径是 /etc/nginx/
启动nginx

service nginx start

关闭nginx

service nginx stop

重载配置

service nginx reload

重启服务

service nginx restart

设置开机启动

systemctl enable nginx

源码安装

源码安装之前需要安装一些依赖库
安装gcc环境

yum -y install gcc gcc-c++

安装pcre

sudo yum -y install pcre pcre-devel # 让 nginx 支持重写功能

安装zlib

# zlib 库提供了很多压缩和解压缩的方式，nginx使用zlib对http包内容进行gzip压缩
yum -y install zlib zlib-devel

安装openssl

# 安全套接字层密码库，用于通信加密
yum -y install openssl openssl-devel

下一步进行nginx安装
下载压缩包并解压

cd /usr/local
wget http://nginx.org/download/nginx-1.10.3.tar.gz
tar -zxvf nginx-1.10.3.tar.gz

解压完成后进入/nginx-1.10.3目录进行源码编译安装到/usr/local/nginx/

cd nginx-1.10.3
./configure --prefix=/usr/local/nginx #检查平台安装环境

编译并安装

make && make install

启动服务

/usr/local/nginx/sbin/nginx
curl localhost #测试访问

源码安装需要把nginx添加到service服务并设置开机启动

进入目录 /usr/lib/systemd/system并新增nginx.service文件

vi /usr/lib/systemd/system/nginx.service

nginx.service文件内容如下

[Unit]
Description=nginx - high performance web server
Documentation=http://nginx.org/en/docs/
After=network.target remote-fs.target nss-lookup.target
   
[Service]
Type=forking
PIDFile=/usr/local/nginx/logs/nginx.pid 
ExecStartPre=/usr/local/nginx/sbin/nginx -t -c /usr/local/nginx/conf/nginx.conf
ExecStart=/usr/local/nginx/sbin/nginx
ExecReload=/usr/local/nginx/sbin/nginx -s reload
ExecStop=/usr/local/nginx/sbin/nginx -s quit
PrivateTmp=true
   
[Install]
WantedBy=multi-user.target

设置开机启动

systemctl enable nginx.service

其它命令

systemctl start nginx.service # 启动
systemctl status nginx.service # 查看状态
systemctl reload nginx.service #重新加载
systemctl stop nginx.service #停止服务
systemctl disable nginx.service #关闭开机启动
